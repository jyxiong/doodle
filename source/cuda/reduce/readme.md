- 从全局内存拷贝到共享内存时顺便计算一次
- io-计算的性价比，可以的话使用小grid，grid-stride，计算多次
- 折半归约

- blocksize作为模板参数，编译期间展开其他循环

性能分析
- 内存连续对齐，warp32各线程一次访问
- 避免分支
- 多使用共享内存和纹理内存这种
- 核函数尽量合并，减少启动的消耗（上下文切换、命令提交、资源分配）
- stream，同时进行计算和IO
- 尽可能提升SM占用率

SM占用率：warp要尽可能的多得在跑
- block大小，影响warp数
    - block太大，单个block消耗资源多，SM能调度的block数变少
    - block太小，warp数不足，不能充分利用SM
- kernel里寄存器、共享内存的使用量
    - 硬件数量有限，你用多了别人就没了
    - 寄存器溢出，变为局部内存

内存
- 寄存器，片上，线程，读写，快
- 共享内存，片上，block，读写，快
- 局部内存，板上，线程，读写，慢
- 常量内存，板上，所有，只读，有缓存/warp广播
- 纹理内存，板上，所有，只读，空间缓存
- 全局内存，板上，所有，读写，慢
- 统一内存，CPU，所有，读写，自动映射
