# 五、CUDA 核心原语与优化技术

## 5.1 `__syncthreads()`

**作用：** 同步 Block 内的所有线程，确保所有线程都执行到该函数后，才继续执行后续代码。

**限制：**
- 仅能在 Block 内部使用，**不能跨 Block 同步**
- **不能在分支语句（`if/else`）内使用**，容易导致死锁

### 死锁场景与规避

**死锁原因：** 部分线程满足条件进入分支并等待在 `__syncthreads()`，其余线程不进入分支，导致永久等待。

**规避方法：** 保证 Block 内所有线程都能执行到 `__syncthreads()`；若必须在逻辑分支内同步，确保所有线程都能进入该分支。

---

## 5.2 跨 Block 全局同步方案

| 方案 | 说明 |
|------|------|
| `cudaDeviceSynchronize()` | 等待所有 Block 执行完毕后再执行后续操作 |
| 全局原子计数器 | 所有 Block 完成后更新计数器，主线程等待计数达到目标值 |
| 多 Stream 同步 | 让不同 Block 所在的 Stream 按顺序执行 |
| CUDA 协作组（Cooperative Groups） | 原生支持跨 Block 同步 |

---

## 5.3 原子操作（Atomic Operations）

**实现原理：** 硬件级的不可中断指令，保证"读-改-写"三步操作一次性完成，通过硬件锁或总线仲裁避免多线程竞争。

**性能开销：** 原子操作导致线程串行执行，开销较高，不宜频繁使用。

### 常见原子操作

| 函数 | 功能 |
|------|------|
| `atomicAdd(addr, val)` | 原子加 |
| `atomicSub(addr, val)` | 原子减 |
| `atomicMax(addr, val)` | 原子取最大值 |
| `atomicMin(addr, val)` | 原子取最小值 |
| `atomicExch(addr, val)` | 原子交换 |
| `atomicCAS(addr, cmp, val)` | 原子比较交换（Compare-And-Swap） |

**适用场景：**
- 全局计数器（统计线程执行次数）
- 多线程对共享变量的更新（归约运算）
- 避免数据竞争的简单写操作

---

## 5.4 Kernel Fusion（内核融合）

**定义：** 将多个独立的小核函数合并为一个大核函数。

**核心原理：**
- 减少全局内存的读写次数（避免中间结果写入全局内存再读取）
- 降低核函数启动开销
- 提升数据局部性

**额外好处：**
- 减少显存占用（无需存储中间结果）
- 简化代码逻辑，减少驱动调度开销
- 提升数据缓存命中率

**典型实例：**
1. 卷积算子 + 激活函数（ReLU）融合
2. 归一化（BatchNorm/LayerNorm）+ 缩放、偏移操作融合
3. 多个逐元素操作（add、mul、relu）融合为一个内核
4. 池化 + 激活函数融合

---

## 5.5 Block Reduce（块内归约）

**定义：** 在单个 Block 内，将所有线程的计算结果归约为一个值（如求和、求最大）。

**实现流程：**
```
1. 线程计算局部结果，写入共享内存
2. __syncthreads() 同步
3. 逐轮合并：相邻线程合并共享内存中的局部结果
4. 最终由一个线程输出 Block 的归约结果
```

**核心优化点：**
1. 消除 Warp Divergence，保证归并过程中线程执行路径一致
2. 避免共享内存 Bank Conflict
3. 循环展开，减少循环开销
4. 充分利用寄存器，减少共享内存访问次数

---

## 5.6 Double Buffer（双缓冲）

**定义：** 使用两组共享内存缓冲区交替工作，实现访存与计算重叠，隐藏访存延迟。

**基本原理：**
```
缓冲区 A → 当前计算使用
缓冲区 B → 同时加载下一批数据
                ↓
完成后交换角色，循环往复
```

**代码实现思路：**
```cpp
__shared__ float bufA[TILE_SIZE];
__shared__ float bufB[TILE_SIZE];

int cur = 0;  // 标志位，0 用 A 计算，1 用 B 计算

// 预加载第一批数据到 bufA
load_data(bufA, ...);
__syncthreads();

for (int i = 1; i <= N; i++) {
    float* compute_buf = (cur == 0) ? bufA : bufB;
    float* load_buf    = (cur == 0) ? bufB : bufA;

    // 异步加载下一批数据（与计算重叠）
    if (i < N) load_data(load_buf, ...);

    // 当前缓冲区执行计算
    compute(compute_buf, ...);

    __syncthreads();
    cur ^= 1;  // 切换缓冲区
}
```

## 5.7 循环展开（Loop Unrolling）

**影响：**
- **优点：** 减少循环控制开销（变量递增、条件判断），增加指令并行度，提升 SM 利用率
- **缺点：** 过度展开会导致寄存器溢出，增加寄存器占用

**合理使用：** 适度展开（4~8 次），结合寄存器占用情况调整，避免过度展开。

---

## 5.8 高效 Element-Wise（逐元素）算子

1. 保证全局内存访存合并，线程索引与数据地址对应
2. 单线程处理多个元素，提升线程利用率
3. 循环展开，减少循环开销
4. 避免分支，消除 Warp Divergence
5. 必要时使用共享内存缓存数据

---

## 5.9 前缀和（Prefix Scan）并行实现

**分块并行思路：**
```
1. 每个 Block 内计算局部前缀和，存储到共享内存
2. 计算所有 Block 局部前缀和的全局前缀和，得到每个 Block 的偏移量
3. 每个 Block 根据全局偏移量调整本地前缀和，得到最终全局前缀和
```

---

## 5.10 并行排序

**分块排序 + 全局归并思路：**
```
1. 每个 Block 内对局部数据进行排序（快速排序、冒泡排序等）
2. 所有 Block 排序完成后，对各 Block 的有序数据进行全局归并
3. 得到最终的有序结果
```
