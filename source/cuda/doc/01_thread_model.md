# 一、CUDA 线程模型

## 1.1 线程层级结构

CUDA 采用三层层级结构：

```
Grid（网格）
  └── Block（线程块）
        └── Thread（线程）
              └── Warp（线程束，硬件最小调度单元，32个线程）
```

| 层级 | 说明 |
|------|------|
| Grid | 核函数启动时的顶层结构，管理所有 Block |
| Block | 线程的集合，块内线程可共享共享内存，支持同步 |
| Thread | 最基本执行单元 |
| Warp | 硬件最小执行与调度单元，固定 32 个线程 |

## 1.2 Block 与 Thread 数量限制

- 一个 Block 最多 **1024 个线程**（主流 GPU，如 A100、3090），且须为 **32 的整数倍**
- 限制来源：SM 的寄存器总量、共享内存容量
- Grid 的大小（Block 数量）无固定硬件上限，由核函数启动参数决定

## 1.3 Warp 基础

- Warp 是 GPU 硬件最小执行单元，固定包含 **32 个线程**
- 同一 Warp 内所有线程同步执行同一条指令（SIMT）
- **32 是硬件设计最优并行粒度**，匹配内存事务大小
- Block 内的线程会被硬件自动划分为若干 Warp；若线程数不是 32 的倍数，多余槽位线程挂起

## 1.4 Warp Divergence（线程束分化）

**定义：** 同一 Warp 内的线程执行不同的分支（如 `if/else`），导致部分线程挂起，分支串行执行。

**影响：**
- 有效指令吞吐量大幅下降
- 原本 32 个线程并行执行的指令，需分多次串行执行，增加执行周期

**避免方法：**
1. 统一 Warp 内所有线程的分支条件
2. 用掩码逻辑（如 `__ballot()`）替代分支判断
3. 对数据重排，让相同分支的线程归为同一 Warp
4. 移除不必要的分支判断

## 1.5 Independent Thread Scheduling（独立线程调度）

- **架构要求：** Volta 及以上（Compute Capability 7.0+）
- **作用：** 允许 Warp 内的线程独立调度、独立执行，无需等待同 Warp 其他线程
- **解决问题：** 同步死锁，提升并行效率

## 1.6 Block 内线程同步

| 方式 | 说明 |
|------|------|
| `__syncthreads()` | 同步 Block 内所有线程（主要方式） |
| 共享内存 | 辅助同步数据可见性 |
| 原子操作 | 保护共享变量的读写 |

## 1.7 跨 Block 同步

> **跨 Block 的线程不能直接同步**，因为不同 Block 可能分配到不同 SM 上执行，SM 间无直接同步机制。

替代方案：
- **内核结束同步：** `cudaDeviceSynchronize()` 等待所有 Block 完成
- 全局原子操作
- 流同步
- CUDA 协作组（Cooperative Groups）
